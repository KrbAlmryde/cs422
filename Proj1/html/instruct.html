Upon loading the page for the first time, the user (YOU) will be presented with a display consisting of three 'walls'. In the center wall you will see two icons which look like the following:

What individuals with typical color vsion would see

The same page in as viewed by individuals who cannot see the color Green

The same page in as viewed by individuals who cannot see the color Red

The same page in as viewed by individuals who cannot see the color Blue

Though the application distinguishes the major categories (Falls vs Finds) by color, Orange and yellow respectively, it does not make the distinction between the different types of meteoroids. In this case, Stony, Iron, and Combined-type. It is up to the user to specifically filter these characteristics using the builtin filter option.

Interaction with the map must be done explicitly with the map directly. That is, the mouse must be hovering over some part of the map itself, otherwise nothing can be interacted with. For example, if the user wishes to zoom into a specific location, the mouse must be hovering over geographical landmass. If the mouse is hovering what could be considered a body of water, interaction is restricted. The same is true when trying to drag the map to reorient the view. Unless the user is dragging landmass, interaction is restricted.

TLDR

  • Does not accommodate color vision deficiency (re not Color Blindness Friendly)
  • icons do not function
  • Lack of visual distinction between Meteor type and source
  • Some interaction issues
